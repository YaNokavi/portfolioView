# Системные требования к фронтенду — portfolioView

## Назначение документа

Документ описывает системные требования к фронтенд-части системы учёта личного портфеля активов. Он определяет технологический стек, архитектуру приложения, маршруты, структуру слоёв, схему взаимодействия с бэкендом и нефункциональные требования. Документ является продолжением бизнес-требований и системных требований бэкенда и должен читаться совместно с ними.

## Маршруты (роутинг)

| Маршрут | Страница | Описание |
|---|---|---|
| `/login` | `AuthPage` | Форма входа (JWT); 2FA если включена на бэке |
| `/` → redirect `/dashboard` | — | Редирект на дашборд |
| `/dashboard` | `DashboardPage` | Главный экран: итоговый баланс, графики, структура портфеля, статусы синхронизации |
| `/categories` | `CategoryPage` | Список всех категорий с их стоимостью, долей, изменением и PnL |
| `/categories/:categoryId` | `CategoryPage` (detail) | Детальная аналитика по выбранной категории: исторический график, подкатегории, активы |
| `/assets` | `AssetPage` | Список всех позиций с фильтрацией и сортировкой |
| `/assets/:assetId` | `AssetPage` (detail) | Детальная карточка актива: цена, PnL, история, средняя цена входа |
| `/allocation` | `AllocationPage` | Целевая аллокация: текущая vs целевая доли, расчёт ребалансировки |
| `/history` | `HistoryPage` | Журнал операций: фильтрация по типу, источнику, дате |
| `/sources` | `SourcesPage` | Управление источниками данных: подключение, статус, последняя синхронизация |
| `/settings` | `SettingsPage` | Настройки аккаунта: базовая валюта, уведомления, управление категориями |

Все маршруты, кроме `/login`, защищены `PrivateRoute` — при отсутствии валидного JWT редирект на `/login`.

## Страницы и их ответственность

### `DashboardPage`

Главный экран. Компонует виджеты без собственной бизнес-логики.

**Виджеты на странице:**
- `BalanceWidget` — суммарная стоимость портфеля в базовой валюте + кнопка смены валюты (USD/RUB)
- `PortfolioChartWidget` — линейный график изменения стоимости с переключателем периода (24h / 7d / 30d / YTD / произвольный)
- `PnLSummaryWidget` — PnL за всё время и за выбранный период
- `CategoryListWidget` — структура портфеля по категориям (pie-chart + таблица)
- `TopPerformerWidget` — крупнейшие позиции (топ-10)
- `SyncStatusWidget` — статусы всех подключённых источников с временем последней синхронизации

**API-зависимости:** `GET /api/portfolio/summary`, `GET /api/portfolio/snapshots`, `GET /api/portfolio/categories`, `GET /api/portfolio/pnl`

---

### `CategoryPage`

Список категорий + детальный режим при клике.

**Список:** стоимость, доля, delta за период, PnL (кроме RUB/USD/USDT/USDC).
**Детальный режим:** исторический график стоимости категории, список подкатегорий, список активов внутри категории.

**API-зависимости:** `GET /api/portfolio/categories`, `GET /api/portfolio/categories/:id`, `GET /api/portfolio/categories/:id/history`

---

### `AssetPage`

Список всех позиций с фильтрацией по категории, источнику, поиску по тикеру.

**Детальная карточка актива:**
- Текущее количество, цена, стоимость
- Средняя цена входа + источник её происхождения (auto / imported / manual) с бейджем
- Unrealised PnL, Realised PnL (если доступен)
- Доходность в %
- История изменения стоимости позиции (линейный график)
- Кнопка «Редактировать» → feature `edit-asset`

**API-зависимости:** `GET /api/portfolio/positions`, `GET /api/portfolio/positions/:id`, `GET /api/portfolio/positions/:id/history`

---

### `AllocationPage`

Настройка и просмотр целевой аллокации.

- Форма задания целевых долей по категориям (в %) и допустимого диапазона отклонения
- Визуализация: текущая доля vs целевая — bar-chart с цветовой индикацией (в норме / вне диапазона)
- Таблица: категория, целевая доля, текущая доля, отклонение в п.п., отклонение в USD, необходимый объём покупки/продажи
- Дисклеймер: «Результат является сценарием ребалансировки, а не инвестиционной рекомендацией»

**API-зависимости:** `GET /api/portfolio/allocation`, `PUT /api/portfolio/allocation`

---

### `HistoryPage`

Журнал операций.

- Таблица операций: дата, тип (покупка / продажа / пополнение / вывод / перевод / проценты / дивиденды / корректировка), инструмент, количество, цена, сумма, источник
- Переводы между своими источниками визуально связаны (иконка «трансфер», не отображаются как доход/расход)
- Фильтры: тип операции, источник, категория, период

**API-зависимости:** `GET /api/portfolio/ledger`

---

### `SourcesPage`

Управление источниками данных.

- Список источников: название, тип, валюта учёта, статус подключения, дата последней синхронизации, технический статус
- Кнопка «Добавить источник» → feature `connect-source` (форма с вводом API-ключа или публичного адреса кошелька)
- Кнопка «Синхронизировать» → feature `manual-sync` (POST запрос, полинг статуса)
- Статус синхронизации с бейджем: OK / ERROR / STALE / PENDING
- При статусе ERROR — отображение текста ошибки из бэкенда

**API-зависимости:** `GET /api/integration/sources`, `POST /api/integration/sources`, `POST /api/integration/sources/:id/sync`

---

### `SettingsPage`

Настройки пользователя.

- Смена базовой валюты портфеля (USD / RUB / другая) — сохраняется на бэкенде
- Управление категориями: просмотр дефолтных (нередактируемые), создание / редактирование / удаление подкатегорий
- Настройки уведомлений (Telegram): токен / chat_id, пороги по отклонению аллокации, изменению стоимости
- Экспорт данных: кнопки экспорта CSV (операции, состав портфеля, история, PnL)

**API-зависимости:** `GET /api/user/settings`, `PUT /api/user/settings`, `GET /api/portfolio/categories`, `POST /api/portfolio/categories`, `GET /api/export/*`

---

### `AuthPage`

Форма входа.

- Поле email + пароль
- Поле кода 2FA (показывается после первого успешного ввода пароля если 2FA включена)
- При успешном логине: сохранение access_token и refresh_token в `localStorage` + редирект на `/dashboard`
- RTK Query `authApi` отвечает за `POST /api/auth/login` и `POST /api/auth/refresh`

## Управление состоянием (Redux)

| Slice / API | Ответственность |
|---|---|
| `authSlice` | Хранение токенов, флага авторизованности, статуса загрузки |
| `portfolioApi` (RTK Query) | Все запросы к `/api/portfolio/*`; кеширование, инвалидация |
| `integrationApi` (RTK Query) | Запросы к `/api/integration/*` (источники, синхронизация) |
| `settingsSlice` | Локальный UI-стейт (выбранная валюта, выбранный период графика) |
| `userApi` (RTK Query) | Запросы к `/api/user/*` (настройки, экспорт) |

Refresh-токен логика: RTK Query `baseQuery` оборачивается в `fetchBaseQueryWithReAuth` — при 401 автоматически вызывается `POST /api/auth/refresh`, при неуспехе — редирект на `/login`.

## Взаимодействие с бэкендом

Фронтенд взаимодействует с бэкендом исключительно через REST JSON API.

**Аутентификация:** Bearer-токен в заголовке `Authorization`. Access-токен хранится в `localStorage`, refresh-токен — в `httpOnly cookie` (если бэкенд поддерживает) или `localStorage` (фолбэк).

**Базовый URL:** настраивается через переменную окружения `REACT_APP_API_URL`.

**Обработка ошибок:** глобальный error-handler в `baseQuery` показывает `Toast`-уведомление при 4xx/5xx. Специфические ошибки (истёкший токен, недоступный источник) обрабатываются на уровне компонента.

**Устаревшие данные:** если бэкенд возвращает поле `dataStale: true` в ответе для портфеля или категории — фронтенд отображает бейдж «Данные могут быть устаревшими» с временной меткой последнего успешного обновления.

## Нефункциональные требования

### Производительность

- Первая загрузка дашборда — не более 2 секунд на типичном канале.
- Code splitting по маршрутам (React.lazy + Suspense) — каждая страница грузится отдельным chunk'ом.
- Мемоизация тяжёлых вычислений (totals, PnL aggregation) через `useMemo` или в RTK Query `transformResponse`.
- Графики с большим числом точек (2 года × ежедневные снапшоты ≈ 730 точек) рендерятся без блокировки основного потока.

### Безопасность

- Чувствительные данные (адреса кошельков, API-ключи) маскируются в UI: показывается только первые 4 и последние 4 символа.
- Токены не логируются в консоль и не попадают в Redux DevTools (применяется `serializableCheck` exclusion для чувствительных полей).
- CSP-заголовки настраиваются на уровне сервера раздачи статики.
- Авторизованные маршруты защищены `PrivateRoute`.

### Доступность и UX

- Приложение является SPA с навигацией без перезагрузки страницы.
- Состояния загрузки отображаются через skeleton-компоненты (не спиннеры на весь экран).
- Все формы валидируются на клиенте через Zod-схемы до отправки на сервер.
- Для числовых значений (цены, PnL, стоимость) применяется форматирование с учётом выбранной базовой валюты: USD — `$1,234.56`, RUB — `1 234,56 ₽`.
- Период графика (24h / 7d / 30d / YTD / custom) сохраняется в URL query params (`?period=7d`) для возможности шаринга и навигации назад.

### Расширяемость

- Добавление новой страницы не требует изменений в существующих модулях (FSD-принцип).
- Добавление нового типа источника данных (нового коннектора) на бэкенде требует только добавления нового варианта в `SourceType` enum и иконки источника в `shared/config`.
- Библиотека графиков (Recharts) заменяема через абстракцию в `shared/ui/Chart` без изменения виджетов.

## Критерии готовности фронтенда MVP

MVP фронтенда можно считать готовым, если пользователь способен:

- [ ] Войти в систему по паролю и выйти из неё
- [ ] Видеть суммарную стоимость портфеля в USD и RUB
- [ ] Видеть график изменения стоимости портфеля за 24h, 7d, 30d, YTD
- [ ] Видеть структуру портфеля по категориям (pie-chart + таблица)
- [ ] Видеть PnL по каждому активу (кроме RUB/USD)
- [ ] Добавить актив вручную через форму
- [ ] Подключить источник данных (Bybit, Т-Инвестиции, кошелёк) и увидеть его статус
- [ ] Запустить синхронизацию вручную и увидеть результат
- [ ] Задать целевую аллокацию и увидеть отклонение в процентах и деньгах
- [ ] Понять, откуда взялась средняя цена входа по активу: из истории операций или из ручного ввода
- [ ] Экспортировать историю операций в CSV
